# GitHub Project Recommendation

Create one GitHub Project named `FrameFlow Roadmap`.

## Suggested fields

- Status
- Milestone
- Area
- Priority
- Size
- Target release
- Decision needed

## Suggested statuses

- Inbox
- Ready
- In progress
- In review
- Blocked
- Done

## Views

- Roadmap by milestone
- Current milestone board
- Architecture decisions
- Bugs and hardening
- Documentation

## Usage rule

Every implementation issue should map to one milestone and one area. Architecture questions should become ADR issues before implementation when they affect long-term direction.
