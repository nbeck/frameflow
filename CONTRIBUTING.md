# Contributing to FrameFlow

Thank you for considering a contribution.

## Before opening a pull request

1. Check existing issues and discussions.
2. For significant changes, open an issue or ADR proposal first.
3. Keep changes focused and milestone-aligned.
4. Run `make check` locally.

## Development setup

```bash
make setup
make check
```

## Pull request expectations

- Clear problem statement
- Tests for behavioral changes
- Documentation updates where relevant
- No unrelated formatting churn
- No secrets, personal photos, or provider credentials

## Architecture changes

Material architecture decisions should be captured as ADRs in `adr/`.
